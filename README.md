# Retailrocket-recommender-system
Проектная работа по курсу ML.Professional OTUS
О наборе данных
Контекст
Набор данных состоит из трех файлов: файла с данными о поведении (events.csv), файла со свойствами товаров (item_properties.сsv) и файла с описанием дерева категорий (category_tree.сsv). Данные были собраны на реальном сайте электронной коммерции. Это необработанные данные, то есть без каких-либо преобразований контента, однако все значения хешированы в целях сохранения конфиденциальности. Цель публикации — стимулировать исследования в области рекомендательных систем с неявной обратной связью.

Содержание
Данные о поведении, то есть о таких событиях, как клики, добавление в корзину, транзакции, отражают взаимодействие пользователей за период в 4,5 месяца. Посетитель может совершить три типа действий: «просмотр», «добавить в корзину» или «транзакция». Всего произошло 2 756 101 событие, включая 2 664 312 просмотров, 69 332 добавления в корзину и 22 457 транзакций, совершенных 1 407 580 уникальными посетителями. Примерно для 90 % событий соответствующие свойства можно найти в файле «item_properties.csv».

Например:

“1439694000000,1, просмотр, 100” означает visitorID = 1, щелкнул элемент с id = 100 в 1439694000000 (временная метка Unix)
“1439694000000,2, транзакция, 1000, 234” означает, что visitorID = 2 приобрел товар с id = 1000 в транзакции с id = 234 в 1439694000000 (временная метка Unix)
Файл со свойствами товаров (item_properties.csv) содержит 20 275 902 строки, то есть различные свойства, описывающие 417 053 уникальных товара. Файл разделен на 2 части из-за ограничений по размеру. Поскольку свойства товара могут меняться со временем (например, цена), каждая строка в файле имеет соответствующую временную метку. Другими словами, файл состоит из объединенных снимков за каждую неделю в файле с данными о поведении. Однако если свойство элемента остается неизменным в течение всего наблюдаемого периода, в файле будет присутствовать только одно значение снимка.
Например, у нас есть три свойства для одного элемента и четыре еженедельных снимка, как показано ниже:

timestamp,itemid,property,value
1439694000000,1,100,1000
1439695000000,1,100,1000
1439696000000,1,100,1000
1439697000000,1,100,1000
1439694000000,1,200,1000
1439695000000,1,200,1100
1439696000000,1,200,1200
1439697000000,1,200,1300
1439694000000,1,300,1000
1439695000000,1,300,1000
1439696000000,1,300,1100
1439697000000,1,300,1100
После слияния снимков это будет выглядеть так:

1439694000000,1,100,1000
1439694000000,1,200,1000
1439695000000,1,200,1100
1439696000000,1,200,1200
1439697000000,1,200,1300
1439694000000,1,300,1000
1439696000000,1,300,1100
Поскольку значение property=100 остается неизменным с течением времени, значение property=200 меняется для каждого снимка, а значение property=300 было изменено один раз.

В файле свойств элементов есть столбец с меткой времени, поскольку все свойства зависят от времени, так как могут меняться с течением времени, например цена, категория и т. д. Изначально этот файл состоял из снимков состояния событий за каждую неделю и содержал более 200 миллионов строк. Мы объединили последовательные значения постоянных свойств, и формат файла изменился с моментального снимка на журнал изменений. Таким образом, постоянные значения будут отображаться в файле только один раз. Это действие значительно сократило количество строк в 10 раз.

Все значения в файле «item_properties.csv», за исключением свойств "categoryid" и "available", были хешированы. Значение свойства "categoryid" содержит идентификатор категории товара. Значение свойства "available" содержит информацию о доступности товара, то есть 1 означает, что товар был в наличии, а 0 — что нет. Все числовые значения в начале помечены символом "n" и имеют точность до трех знаков после запятой, например, "5" станет "n5.000", "-3.67584" станет "n-3.675". Все слова в текстовых значениях были нормализованы (процедура стемминга: https://en.wikipedia.org/wiki/Stemming) и хешированы, числа были обработаны как указано выше, например, текст «Привет, мир, 2017!» превратится в «24214 44214 n2017.000».

В файле дерева категорий 1669 строк. В каждой строке файла указан дочерний идентификатор категории и соответствующий родительский идентификатор.
Например:

Строка «100,200» означает, что у categoryid=1 есть родитель с categoryid=200
Строка «300» означает, что у categoryid нет родителя в дереве

Retail Rocket (retailrocket.io) помогает интернет-покупателям принимать более взвешенные решения о покупках, предоставляя персонализированные рекомендации в режиме реального времени через различные каналы. У компании более 100 миллионов уникальных пользователей в месяц и более 1000 розничных партнеров по всему миру.
